RELATÓRIO DO PROJETO RPG – VALE-CINZENTO
Estrutura de Classes
O projeto foi desenvolvido com base em uma hierarquia de herança bem definida.
A classe abstrata Personagem concentra os atributos e comportamentos comuns (nome, HP, ataque, defesa e nível), servindo como base para as subclasses Guerreiro, Mago, Arqueiro e Inimigo, que especializam esses elementos conforme suas funções no jogo.
Além das classes principais, foram utilizadas classes auxiliares para organização e suporte do sistema:
– Item: representa objetos colecionáveis como poções, troféus e itens com efeitos especiais.
– Inventario: gerencia os itens acumulados, controlando adições, remoções e quantidades.
– TipoEfeito (enum): classifica os tipos de efeitos disponíveis, como cura e aumento de atributos.
Recursos e Métodos Implementados
Foram aplicados métodos-chave da linguagem Java para garantir uma experiência funcional e coesa:
– toString(): fornece uma descrição legível dos objetos.
– equals() e hashCode(): permitem comparar itens e utilizá-los corretamente em coleções.
– clone(): realiza cópias profundas, evitando o compartilhamento de referências.
– compareTo(): utilizado para ordenar o inventário de forma alfabética.
O resultado final é um jogo funcional, com variações sutis de jogabilidade que tornam cada jornada única. O sistema de combate e progressão foi desenvolvido para ser fluido, com elementos de acaso que influenciam o desenrolar das batalhas. A narrativa do Vale-Cinzento adiciona imersão à experiência, incluindo um final secreto que premia jogadores atentos às decisões tomadas.
